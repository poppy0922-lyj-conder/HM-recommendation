# SHAP 模型可解释性分析报告

## 概述

本报告基于 SHAP (SHapley Additive exPlanations) TreeExplainer 对 H&M 推荐模型的 56 维特征进行可解释性分析。SHAP 基于博弈论中的 Shapley 值，为每个特征分配其对模型预测的贡献值，具有数学上的公平性和一致性。

### 实验设置

- **模型**: LightGBM LambdaRank (56维: 23基线 + 32 a2v + 1 v2v_sim)
- **SHAP 数据**: 127,515 行 × 56 维 (holdout 验证集, 随机采样 2000 用户)
- **正样本率**: 0.20% (高稀疏性，符合推荐场景特征)

### 1.3 关键发现

1. **popularity_score 统治力极强**: 单个特征贡献 (0.7089) 远超其他任何特征，说明商品流行度是推荐排序的最核心信号。这是推荐系统中常见的"马太效应"——热门商品天然更容易被推荐。

2. **a2v 嵌入整体贡献最大**: 32 维 a2v 嵌入的分组贡献 (1.0079) 超过商品特征 (0.8684)，虽然每维贡献较小 (均值 0.0315)，但整体汇聚后成为模型最强的信号来源。这验证了 Word2Vec 序列嵌入的有效性。

3. **v2v_sim 是最高效的单维特征**: 单维贡献 0.1927，仅次于 popularity_score。它衡量用户历史购买商品与候选商品在嵌入空间中的余弦相似度，本质上是"用户偏好向量"与"商品向量"的匹配度。

4. **交互特征 (3 维) 效率最高**: 平均每维 0.0806，远超 a2v 嵌入的 0.0315。用户-商品层面的交互信号 (购买次数、最近购买时间) 是非常强力的个性化信号。

5. **客户特征贡献最低**: 4 维客户特征合计仅 0.1799，说明相对于商品和交互信号，客户画像 (年龄、邮编等) 的预测能力有限。

## 2. 特征依赖关系

### 2.1 popularity_score 与购买概率

popularity_score 呈现**单调正相关**: 商品流行度越高，SHAP 值越大 (正向推动推荐)。高分区域呈线性递增关系，说明模型充分利用了流行度的排序能力。但在极低流行度区域存在"长尾"，部分低流行商品仍能获得正向推荐 (可能来自用户个性化历史匹配)。

### 2.2 v2v_sim 余弦相似度

v2v_sim (用户历史嵌入均值与候选商品嵌入的余弦相似度) 呈现**正相关关系**，阈值约在 0.80 附近。高于此阈值的商品被显著正向推动，低于此则贡献接近零或负向。说明模型学会了一个清晰的"语义匹配边界"——只有当候选商品与用户历史偏好足够相似时，才给予正向信号。

### 2.3 a2v 嵌入维度分析

各 a2v 维度呈现不同的模式：
- **a2v_10**: 双侧贡献，在高值和低值区域都有正向 SHAP，说明该维度编码了一个二分类的商品属性 (例如是否属于某个品类)
- **a2v_27**: 偏正向高值贡献，逐渐上升
- **a2v_24**: 呈现清晰的分段线性模式

这反映了 Word2Vec 各维度在无监督训练后自然形成了对商品不同属性的编码，模型自动学习了对各维度的非线性响应模式。

### 2.4 last_buy_days / R_days

- **last_buy_days** (用户上次购买某商品的天数): 呈现**单调负相关**。购买时间越近 (天数越小)，正向贡献越大；购买时间越久 (天数越大)，负向贡献越强。这是典型的"近期偏好" (recency bias) 信号。
- **R_days** (用户最近一次购买距今的天数): 类似模式，反映了用户的活跃度。

### 2.5 cf_score (协同过滤得分)

cf_score 呈现**正相关但非线性**: 较高分数带来正向贡献，低分数区域 SHAP 值稳定在负值区间。说明 Item-CF 作为召回通道的排序信号有效，高分商品被强烈推荐，而低分商品则被抑制。

## 3. 典型用户案例分析

### 3.1 低频用户 (偶尔购买者)

- **主要推动因素**: popularity_score (商品流行度)
- **主要抑制因素**: a2v 嵌入 (历史购买太少，语义匹配力不足)
- **特征**: 由于历史交互极少，模型更多地依赖全局流行度进行推荐，个性化信号较弱

### 3.2 多样性用户 (跨品类购买者)

- **主要推动因素**: popularity_score + v2v_sim (流行度 + 语义匹配)
- **主要抑制因素**: 部分 a2v 维度负向
- **特征**: 跨品类购买增加了语义空间中的"偏好范围"，v2v_sim 依然能有效匹配

### 3.3 高频复购用户 (忠实购买者)

- **主要推动因素**: last_buy_days + buy_count + v2v_sim
- **主要抑制因素**: popularity_score (相对弱化)
- **特征**: 历史交互丰富的用户，模型明显更依赖**个性化信号** (交互特征 + 语义匹配)，而非全局流行度。这是推荐系统理想的"冷热分离"效果

## 4. 高分 vs 低分样本对比

将预测得分最高的 10% 与最低的 10% 样本进行对比：

| 特征 | 高分 \|SHAP\| | 低分 \|SHAP\| | 差值 | 含义 |
|:----:|:----------:|:----------:|:----:|:----:|
| popularity_score | 0.645 | **2.007** | -1.361 | 低分样本极度依赖流行度 (说明这些商品虽然流行但不符合个性化需求) |
| last_buy_days | **0.457** | 0.107 | **+0.350** | 高分样本有显著更强的"近期购买"信号 |
| first_buy_days | **0.109** | 0.020 | **+0.089** | 高分样本的首次购买特征更有区分度 |
| R_days | **0.123** | 0.053 | **+0.070** | 高分样本用户更活跃 (R_days 更小) |
| buy_count | **0.123** | 0.061 | **+0.062** | 高分样本有更多历史购买次数 |
| cf_score | **0.126** | 0.073 | **+0.053** | 高分样本的协同过滤得分更高 |
| v2v_sim | **0.202** | 0.171 | +0.030 | 高分样本语义匹配度略高 |

### 关键洞察

- **低分样本的"流行度陷阱"**: 低分样本的 popularity_score 贡献 (2.007) 远超高分样本 (0.645)，说明低分商品虽然流行度高但仍然被预测为负。这意味着模型学会了区分"流行但不相关"——用户不感兴趣的热门商品。

- **个性化信号是高分的关键**: 高分样本中 last_buy_days (+0.350)、buy_count (+0.062)、cf_score (+0.053) 等个性化信号显著更强。模型对有交互历史的用户能做出更精准的预测。

- **冷启动挑战**: 低分样本缺乏交互信号，模型退化为"流行度预测"，这与冷启动用户的推荐困境一致。

## 5. 结论与建议

### 5.1 模型行为总结

1. **双通道机制**: 模型同时使用"流行度通道" (popularity_score) 和"个性化通道" (交互特征 + 语义嵌入)，根据用户历史丰富度动态调整权重
2. **语义嵌入有效**: 32 维 a2v + 1 维 v2v_sim 合计贡献 1.2006，是模型最重要的信号来源组
3. **时间衰减感知**: last_buy_days、first_buy_days、R_days 多个时间特征表明模型对"时间衰减"有清晰的建模能力

### 5.2 优化方向

1. **增强语义嵌入**: a2v 嵌入整体贡献最高但每维较分散，可通过增加 vector_size 或改进训练方式 (如加入商品属性约束) 提升各维度的信息密度
2. **优化冷启动**: 低分样本分析显示冷启动用户缺乏个性化信号，可考虑加入用户画像增强或基于内容的推荐兜底
3. **特征压缩**: 32 维 a2v 中部分维度贡献极低，可通过特征选择压缩到 16-24 维，降低模型复杂度
4. **交叉特征**: 当前特征均为线性相加，可尝试交互特征 (如 popularity_score × v2v_sim) 来建模更复杂的推荐模式
